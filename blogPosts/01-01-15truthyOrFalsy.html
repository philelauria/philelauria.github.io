== vs === in JavaScript
January 1st, 2015

I thought it might be useful to make a post as a quick reference between the == (abstract comparison) and === (strict comparison) operators. It's typically taught that the best practice is to default to ===. The essential difference is that == does a type conversion to each of the values prior to making the comparison. As a result, using == opens up the comparison to resulting in a true value in many more scenarios.
